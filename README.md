# RetinaScreen AI

AI-powered diabetic retinopathy detection from retinal fundus images.

A complete system with three layers:
1. **Training Pipeline** — config-driven ML training with augmentation, metrics, and experiment tracking
2. **Inference API** — FastAPI server serving ONNX models via REST endpoints
3. **Deployment** — Docker-ready with CPU/GPU support

## Architecture Overview

```
                    ┌──────────────────────────────────────────┐
                    │           Training Pipeline              │
                    │                                          │
  Kaggle/URL ─────► │ Data Ingestion ─► Validation ─► Split   │
                    │                                          │
                    │ Augmentation ─► Trainer ─► Evaluation    │
                    │                              │           │
                    │                  Visualization │          │
                    └──────────────────────────────┼───────────┘
                                                   │
                                          ONNX Export
                                                   │
                                                   ▼
                    ┌──────────────────────────────────────────┐
                    │           FastAPI Inference Server       │
                    │                                          │
  User Upload ───► │ /api/v1/diagnose ─► Preprocess ─► ONNX   │
                    │                              │           │
                    │                   Softmax ─► Response    │
                    └──────────────────────────────────────────┘
```

The system works in two independent phases:

**Phase 1: Training** — Download or point to a dataset, validate and split it, train an EfficientNet model with augmentations, export to ONNX.

**Phase 2: Inference** — Load the ONNX model into a FastAPI server, accept retinal images via HTTP POST, return diagnosis with confidence scores.

## Training Pipeline

### Dataset Format

The pipeline expects:
- A directory of retinal fundus images (`.jpg`, `.jpeg`, `.png`, `.tiff`, `.tif`)
- A CSV file with columns: `image` (filename), `diagnosis` (integer 0–4)

Example `train.csv`:

```csv
image,diagnosis
0001_left.jpg,0
0002_right.jpg,2
0003_left.jpg,4
```

### DR Severity Grades (0–4)

| Class | Label | Description |
|-------|-------|-------------|
| 0 | No DR | Healthy retina |
| 1 | Mild NPDR | Mild non-proliferative DR |
| 2 | Moderate NPDR | Moderate non-proliferative DR |
| 3 | Severe NPDR | Severe non-proliferative DR |
| 4 | Proliferative DR | Advanced, vision-threatening DR |

### Running the Pipeline

#### Full pipeline (auto-download from Kaggle + train)

```bash
python pipeline/run_pipeline.py
```

This requires the `kagglehub` package and internet access. The default config downloads the APTOS 2024 dataset.

#### Use local data instead

Edit `pipeline/config/config.yaml`:

```yaml
data:
  source: "local"          # ← change from "kaggle" to "local"
  local_path: "data/images" # ← your image directory
  label_file: "data/train.csv"  # ← your CSV with image,diagnosis columns
```

Then run:

```bash
python pipeline/run_pipeline.py
```

#### Download and validate only

```bash
python pipeline/run_pipeline.py --download-only
```

Downloads data, validates images (checks for corruption, size, format), plots label distribution, and exits.

#### Evaluate a trained checkpoint

```bash
python pipeline/run_pipeline.py --eval-only
```

Loads `outputs/checkpoints/model_best.pt` and runs it on the test set.

#### Custom config file

```bash
python pipeline/run_pipeline.py --config my_config.yaml
```

### What the Pipeline Does (Step by Step)

| Step | Description | Output |
|------|-------------|--------|
| 1. Data Ingestion | Downloads from Kaggle/URL or uses local files | Images in `data/images/` |
| 2. Data Validation | Checks each image: format, corruption, min size, aspect ratio | Cleaned CSV, label distribution plot |
| 3. Data Splitting | Stratified 75/15/10 split preserving class balance | `train.csv`, `val.csv`, `test.csv` |
| 4. Augmentation | Albumentations: flips, rotation, brightness, hue, scale | On-the-fly during training |
| 5. Training | EfficientNet-B4 with AdamW, cosine LR, mixed precision, early stopping | Checkpoints in `outputs/checkpoints/` |
| 6. Evaluation | Accuracy, precision, recall, F1, quadratic kappa, ROC AUC | Metrics printed + saved to JSON |
| 7. Visualization | Loss curves, validation metrics, confusion matrix | PNG files in `outputs/plots/` |

### Export to ONNX

After training, convert the best checkpoint to ONNX format:

```bash
python pipeline/export_model.py
```

Output: `outputs/exports/model.onnx`

Copy it to the API's expected location:

```bash
copy outputs\exports\model.onnx data\models\model.onnx
```

Then restart the API to use the trained model.

### Configuration File Reference

`pipeline/config/config.yaml`:

```yaml
data:
  source: "kaggle"                    # "kaggle", "local", or "url"
  kaggle_dataset: "aifymag/aptos-2024-blindness-detection"
  local_path: "data/images"
  label_file: "data/train.csv"
  img_size: 512                       # Input image size (pixels)
  val_split: 0.15                     # Fraction for validation
  test_split: 0.10                    # Fraction for testing
  seed: 42                            # Random seed for reproducibility
  apply_clahe: true                   # Apply CLAHE contrast enhancement
  clahe_clip_limit: 2.0
  clahe_grid_size: 8

augmentation:
  train:
    horizontal_flip: true
    vertical_flip: false
    rotation_limit: 30                # Degrees
    brightness_contrast: [0.8, 1.2]   # [brightness_lower, contrast_upper]
    hue_saturation: [0.0, 0.1]        # [hue_shift, sat_shift]
    scale_limit: 0.15                 # Random scale ±15%
  eval:
    horizontal_flip: false

model:
  architecture: "efficientnet_b4"     # Architecture to use
  pretrained: true                    # Start from ImageNet weights
  num_classes: 5                      # DR severity grades (0-4)
  dropout: 0.3                        # Dropout rate

training:
  batch_size: 16                      # Batch size (reduce if OOM)
  epochs: 50                          # Max epochs
  lr: 0.0001                          # Learning rate
  weight_decay: 0.01                  # AdamW weight decay
  scheduler: "cosine"                 # "cosine", "step", or "plateau"
  early_stop_patience: 10             # Stop if no improvement for N epochs
  gradient_clip: 1.0                  # Max gradient norm
  label_smoothing: 0.1                # Label smoothing for regularization
  mixed_precision: true               # Use FP16 on GPU
  workers: 4                          # Data loader workers

logging:
  use_mlflow: false                   # Enable MLflow tracking
  mlflow_tracking_uri: "outputs/mlruns"
  experiment_name: "retinascreen"
  save_every_n_epochs: 5              # Save checkpoint every N epochs

export:
  onnx: true
  onnx_opset: 17
  output_dir: "outputs/exports"
```

## Inference API

### Starting the Server

```bash
# Development (with auto-reload)
python -m uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000

# Production
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

The server automatically loads the ONNX model from `data/models/model.onnx` on startup.

### Model Requirements for the API

The API expects an ONNX model at `data/models/model.onnx` with:
- **Input**: `"input"`, shape `(batch, 3, height, width)`, dtype `float32`
- **Output**: `"output"`, shape `(batch, 5)`, dtype `float32`
- Image size: 512×512 pixels (configurable)

The pipeline's export script (`pipeline/export_model.py`) produces a compatible model.

### API Endpoints

#### `GET /`

Root endpoint. Returns service metadata and available endpoints.

**Response:**
```json
{
  "service": "RetinaScreen AI",
  "version": "1.0.0",
  "endpoints": {
    "diagnose": "POST /api/v1/diagnose",
    "health": "GET /api/v1/health",
    "labels": "GET /api/v1/labels"
  }
}
```

#### `POST /api/v1/diagnose`

Upload a retinal fundus image for diagnosis.

**Request:** `multipart/form-data` with field `file`

| Parameter | Type | Description |
|-----------|------|-------------|
| file | image | JPG, PNG, or TIFF. Max 10 MB. |

**Response:**
```json
{
  "filename": "retina.jpg",
  "predictions": [
    {"class_id": 0, "label": "No DR", "confidence": 0.85},
    {"class_id": 2, "label": "Moderate NPDR", "confidence": 0.10},
    {"class_id": 1, "label": "Mild NPDR", "confidence": 0.03},
    {"class_id": 3, "label": "Severe NPDR", "confidence": 0.01},
    {"class_id": 4, "label": "Proliferative DR", "confidence": 0.01}
  ],
  "primary_diagnosis": {"class_id": 0, "label": "No DR", "confidence": 0.85},
  "processing_time_ms": 56.48
}
```

#### `GET /api/v1/health`

Health check endpoint.

**Response:**
```json
{
  "status": "ok",
  "model_loaded": true,
  "version": "1.0.0"
}
```

#### `GET /api/v1/labels`

Returns the class labels used by the model.

**Response:**
```json
{
  "classes": [
    "No DR",
    "Mild NPDR",
    "Moderate NPDR",
    "Severe NPDR",
    "Proliferative DR"
  ]
}
```

### Example: Diagnose via cURL

```bash
curl -X POST http://localhost:8000/api/v1/diagnose \
  -F "file=@retina_scan.jpg" | python -m json.tool
```

### Example: Diagnose via Python

```python
import requests

url = "http://localhost:8000/api/v1/diagnose"
files = {"file": open("retina_scan.jpg", "rb")}
resp = requests.post(url, files=files)
result = resp.json()

print(f"Diagnosis: {result['primary_diagnosis']['label']}")
print(f"Confidence: {result['primary_diagnosis']['confidence']:.2%}")
for p in result['predictions']:
    print(f"  {p['label']}: {p['confidence']:.2%}")
```

### Image Preprocessing (API side)

When a user uploads an image, the API performs these steps:

1. **Validate** — checks file extension (`.jpg`, `.jpeg`, `.png`, `.tiff`, `.tif`) and file size (max 10 MB)
2. **Load** — reads the image with OpenCV, converts BGR → RGB
3. **Resize** — resizes to 512×512 pixels using `INTER_AREA` interpolation
4. **Normalize** — scale pixel values to [0,1], then apply ImageNet mean/std normalization (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
5. **Transpose** — HWC → CHW format
6. **Add batch dimension** — shape becomes `(1, 3, 512, 512)`
7. **Run ONNX inference** — feeds the tensor through the model
8. **Softmax** — converts logits to probabilities
9. **Format response** — sorts predictions by confidence, returns primary diagnosis

## FAQ / Troubleshooting

### "No module named 'onnx'"

Install dependencies:
```bash
pip install -r requirements.txt
```

### "ONNX model not found"

You need a model at `data/models/model.onnx`. Either:
- Run `python setup_dev.py` for a dummy test model
- Run `python pipeline/export_model.py` after training
- Download a pretrained model and place it there

### "CUDAExecutionProvider is not available"

This is normal on CPU-only machines. The server falls back to CPU automatically. Training will also fall back to CPU (but will be slow).

### "Out of memory during training"

Reduce `batch_size` in `config.yaml` (try 8 or 4). Also reduce `workers` to 0 or 2.

### "File too large" error

The API accepts images up to 10 MB. Resize your image before uploading.

### "Unsupported file type"

Allowed formats: `.jpg`, `.jpeg`, `.png`, `.tiff`, `.tif`. Convert your image to one of these formats.

### "Cannot load image" / "Corrupt or unreadable"

The image file may be damaged. Run `python pipeline/run_pipeline.py --download-only` to validate your dataset.

### How do I use a different model architecture?

Edit `config.yaml`:
```yaml
model:
  architecture: "resnet50"  # or "densenet121", "efficientnet_b0", etc.
```

Supported architectures: any from `torchvision.models` (EfficientNet, ResNet, DenseNet, etc.).

### How do I deploy on a GPU server?

The API auto-detects CUDA. For the Docker deployment, use NVIDIA's container toolkit:

```yaml
# In docker-compose.yml, add:
# deploy:
#   resources:
#     reservations:
#       devices:
#         - driver: nvidia
#           count: 1
#           capabilities: [gpu]
```

### Where are the test results saved?

- `outputs/training_results.json` — full training run results
- `outputs/test_metrics.json` — evaluation-only results
- `outputs/plots/` — visualizations (loss curves, confusion matrix, etc.)

### How to run tests?

